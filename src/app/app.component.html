Modelos de Referencia

Introducción

Origen y objetivos

A principios de los 70, se desarrollan arquitecturas de red en capas.

Los Modelos de referencia consisten en la definición de las funciones de cada nivel y los interfaces.

Arquitectura en capas

¿En qué consiste?

Agrupar funciones en grupos relacionados.

Las funciones se dividen en subconjuntos, denominados capas.

Protocolos

  • Semántica: se refiere al significado de cada sección de bits.
  • Sintaxis: incluye aspectos como el formato de datos.
  • Temporización: incluye la secuencación del mensaje.

Encapsulamiento

Cada capa usa servicios de la capa inferior para poder transferir sus PDUs

Primitivas

  • 1. Request
  • 2. Indication
  • 3. Response
  • 4. Confirmation

Ventajas

  • Simplificación del diseño.
  • Facilidad en las modificaciones del diseño.
  • Partición del diseño.
  • Interoperatividad.

Modelo OSI

Organización de los niveles

Sistemas intermedios y terminales

Sistemas terminales: Proporcionan funcionalidad completa desde las capas inferiores hasta las aplicaciones del usuario.

Sistemas intermedios: Hacen posible la interconexión entre equipos terminales no adyacentes, implementando las funciones de conmutación y enrutamiento.

Niveles del Modelo OSI

Nivel físico

Coordina las funciones necesarias para transmitir el flujo de datos sobre un medio físico.

Funciones

  • Características físicas de las interfaces y el medio.
  • Define el tipo de señal que se utilizará en la transmisión.
  • Define el tipo de codificación.
  • Tasa de datos.
  • Sincronización de los bits.
  • Configuración de la línea.
  • Topología física.
  • Modo de transmisión.

Nivel de enlace de datos

Transforma el nivel físico, un simple medio de transmisión, en un enlace fiable.

Hace que el nivel físico aparezca ante el nivel superior como un medio libre de errores.

Funciones

  • Entramado.
  • Direccionamiento físico.
  • Control de flujo.
  • Control de errores.
  • Control de acceso.

Nivel de red

Responsable de la entrega de un paquete desde el origen al destino.

Funciones

  • Direccionamiento lógico.
  • Encaminamiento.

Nivel de transporte

Responsable de la entrega extremo a extremo del mensaje completo desde un proceso a otro.

Funciones

  • Direccionamiento en punto de servicio.
  • Segmentación y reensamblado.
  • Control de conexión.
  • Control de flujo.
  • Control de errores.

Nivel de sesión

Se encarga de autenticar al usuario y se puede configurar de forma que pueda recuperar el estado de la comunicación en caso de que ocurra algún problema.

Funciones

  • Control de diálogo.
  • Sincronización.

Nivel de presentación

Está relacionado con la sintaxis y la semántica de la información intercambiada entre los dos sistemas.

Funciones

  • Representación.
  • Cifrado.
  • Compresión.

Nivel de aplicación

Permite al usuario acceder a la red.

Funciones

  • Terminal virtual de red.
  • Transferencia, acceso y gestión de archivos.
  • Servicio de correo.
  • Servicios de directorio.

Equipos de interconexión

Repetidor

Regenera la señal para ampliar una red local y compensa atenuación y distorsión de la señal.

Bridge

Repetidor inteligente que filtra el tráfico que pasa de un puerto a otro.

Gateway

Dispositivo inteligente que actúa a nivel de red o superior y traduce entre dos dominios

Router

Gateway capaz de decidir no solo el puerto de salida, sino cuáles serán los nodos subsiguientes de la ruta que deberá seguir el mensaje hasta alcanzar su destino.

Switch

Interconecta dos o más host de manera similar a los bridges, pasando datos de un segmento a otro.
